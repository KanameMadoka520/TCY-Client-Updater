## 核心技术架构与原理

本项目采用轻量级混合开发模式，引入了 **"序列化队列"**、**"智能补票"** 与 **"High-DPI 自适应"** 架构：

### 1. 前后端分离与主动握手

* **前端 (View)**: `index.html` 负责界面交互。全面采用 **SVG 矢量图标**，引入 **动态粒子系统 (Particle System)** 与 **CSS3 故障风 (Glitch)** 特效，提升视觉专业度。
* **后端 (Model)**: `TCYServer_MCUpdater.py` 负责核心逻辑（文件IO、系统调用、Win32 API交互）。
* **通信机制**: 采用 **前端主动握手** 模式。前端 DOM 加载完毕后通知后端，后端再注入初始化数据，解决界面"未响应"或死锁问题。

### 2. 骨肉分离与动态加速

* **骨 (Skeleton)**: 极小的 `update.zip`（仅含配置、脚本、清单），托管在 GitHub/Cloudflare，实现秒级响应。
* **肉 (Flesh)**: 大体积模组文件（Jar）托管于 GitHub Release，通过清单索引。
* **动态镜像**: 针对 CN 线路，后端会自动拦截 GitHub 请求并动态拼接镜像前缀（如 `gh-proxy`），实现国内外双线高速更新。

### 3. 序列化状态同步

* **智能队列机制**: 更新不再是简单的"覆盖最新版"。系统会自动计算你当前版本与最新版本之间**缺失的所有更新**，并将它们构建为一个按时间排序的队列。
* **批量原子化执行**: 用户确认后，程序会自动**依次**下载并安装队列中的每一个版本，确保没有任何中间补丁被遗漏（告别"跳版本"导致的文件缺失）。

### 4. 双轨版本控制机制

* **启动器自更新**: 支持语义化版本（如 `1.0.7`），采用 UTF-8 批处理脚本，**自适应保留用户重命名后的文件名**（即使是包含 Emoji 的怪异名字）。
* **内容更新**: 采用时间戳版本（如 `26.02.06.15.53`），支持增量更新与可选内容的"补票"机制。

### 5. 高分屏适配与配置持久化

* **High-DPI 完美适配**: 后端通过 `windll.user32` 直接调用 Win32 API，自动计算物理像素与逻辑像素的比例。无论是在 1080P 还是 4K 屏幕下，窗口大小调整均精确到像素级，避免模糊。
* **JSON 存储**: 所有设置（背景、模糊度、字体、分辨率预设、镜像源）及跳过的版本记录均持久化存储。

### 6. 原子性更新与回滚 (v1.0.3 新增)

* **两阶段原子更新**: 所有文件先下载到 `temp_staging/` 暂存区并完成完整性校验（SHA256 或 size），校验全部通过后才执行备份旧文件 → 删除/复制 actions → 从暂存区移入目标位置的应用操作。任何阶段失败自动回滚。
* **自动备份与回滚**: 更新前自动将受影响文件备份到 `.update_backups/backup_<版本号>/`，更新失败自动恢复。支持手动回滚和最大备份数配置（1-5）。

### 7. 更新前摘要预览与确认门禁 (v1.0.5 新增)

* **Token 校验门禁**: 用户点击更新后，系统先生成一份更新摘要（目标版本、文件数量、预估总下载量、影响路径），并产生一个有效期 600 秒的 `plan_token`。用户必须在摘要界面明确点击"继续更新"才会开始下载，未确认不会产生任何下载副作用。
* **取消安全**: 用户选择取消后，token 作废，不新增临时文件，不进入下载中状态。

### 8. 断点续传可行性评估与条件启用 (v1.0.5 新增)

* **自动评估**: 下载前对每个 URL 发起 HEAD/Range 探测，检测镜像是否支持 HTTP Range、Content-Length 是否可信、是否有 ETag/Last-Modified 等条件。
* **条件启用**: 仅当服务器完整支持 Range 请求且文件大小可知时才启用续传，否则自动回退全量下载。
* **透明日志**: 评估结论、关键原因均写入活动日志，方便排查。

## 功能特性

### 🛡️ 安全与校验 (Safety & Verification)

* **启动前置强校验**:
    * 程序启动时会自动进行深度目录扫描。
    * **严格路径检测**: 强制检查相对路径 `.minecraft/versions/异界战斗幻想` 是否存在。
    * **防呆拦截**: 若未检测到正确路径（如用户将更新器误放在桌面），将弹出警告窗口阻断进入，防止文件解压错乱（支持知道自己在做什么的用户强制跳过）。
* **SHA256 文件完整性校验** (v1.0.3 新增):
    * `manifest.json` 的 `external_files` 支持可选 `sha256` 字段。
    * 下载完成后自动校验哈希，不匹配则删除重下（最多重试 2 次）。
    * 向后兼容：没有 `sha256` 字段时退回到原有的 size 检查。

### 🛡️ 智能更新系统 (Smart Update)

* **更新前摘要预览** (v1.0.5 新增):
    * 开始下载前，弹出摘要卡片展示：目标版本列表、待下载文件数量、预估总下载大小、受影响路径。
    * 用户必须明确点击"继续更新"或"取消"，未确认时不会开始下载。
    * 摘要带有 600 秒有效期的 token 校验，超时需重新获取。
* **断点续传** (v1.0.5 新增):
    * 下载前自动探测镜像是否支持 HTTP Range 请求。
    * 条件满足时启用续传（从已下载位置继续），条件不满足时自动回退全量下载。
    * 续传仅优化下载阶段，最终仍以完整校验（SHA256/size）通过才进入应用阶段。
* **可视化进度反馈**:
    * 实时显示 **下载速度 (MB/s, KB/s)**。
    * 精确显示当前正在处理的文件名及该文件的下载百分比。
    * **单文件详细进度** (v1.0.3 新增): 显示文件级进度（已下载 / 总大小）、总进度（第 X / Y 个文件）、预计剩余时间 (ETA)。
* **并行下载** (v1.0.3 新增):
    * 使用 `ThreadPoolExecutor` 多线程并行下载 `external_files`。
    * 并发数可配置（默认 3，最大 6），在设置页面调整。
    * 严格限制在当前版本更新范围内，每个版本更新仍按顺序执行。
    * 进度回调使用线程锁保证线程安全。
* **可选更新与补票**:
    * **可选 (Optional)**: 针对非强制性内容（如辅助模组），提供复选框，可选择跳过。
    * **补票机制 (Catch-up)**: 系统会记住你跳过的可选更新。下次检查时，它依然会出现在列表中提醒你"补票"，直到安装为止。
* **强制更新锁定**: 涉及服务器连接的核心更新将被锁定为"必选"。
* **强制拉取历史版本**:
    * 更新页面提供"强制拉取历史版本"按钮，用于你明确知道操作目的、想要强制重新应用某一次历史更新的情况。
    * 无需手动修改 `launcher_settings.json` 中的版本号，直接从列表中选择目标版本即可。
    * 点击后会列出所有历史版本供单选，选定后再点击更新才会执行，不会误触。
* **综合版本信息弹窗**: 检查更新时，客户端版本信息与更新器版本信息统一在同一弹窗中展示，同时显示各来源地址的获取结果。
* **更新回滚** (v1.0.3 新增):
    * 更新前自动备份受影响文件，更新失败自动回滚。
    * 更新页面增加"手动回滚"区域，可查看备份列表并一键恢复。
    * 设置中可配置"最大备份数"（默认 1，最大 5），超出自动删除最旧备份。
* **本地更新包安装** (v1.0.3 新增):
    * 更新页面增加"选择本地更新包 (ZIP)"按钮。
    * 选择本地 zip 文件后先读取内部 `manifest.json` 预览更新内容。
    * 用户确认后执行安装，复用原子性更新流程。

### 📦 Config 备份与还原 (v1.0.5 新增)

玩家可以对 `config/` 目录的指定子目录和根级文件进行命名备份，并在还原前预览覆盖范围：

* **可选范围备份**:
    * 进入 Config 页面后，可勾选要备份的子目录和根级文件（如 `.cfg`、`.json` 等配置文件）。
    * 输入备份名称后一键创建，备份完成后立即出现在列表中。
    * 每个备份记录包含元数据：名称、创建时间、覆盖范围。
* **还原前预览**:
    * 选择某个备份后，系统展示将被覆盖的目录/文件清单。
    * 必须勾选确认复选框并点击确认后才会执行还原。
* **还原前自动安全备份**:
    * 执行还原操作前，系统自动创建一份"还原前备份"（pre-restore safety backup），防止误操作导致无法恢复。
    * 还原完成后 config 内容与所选备份一致。
* **搜索过滤**:
    * 备份选择界面支持搜索过滤，快速定位目标文件夹/文件。

### 🧩 Mod 管理器

* **本地元数据读取**: 读取 `.jar` / `.jar.disabled` 文件，解析 `fabric.mod.json` 或 `mods.toml`，展示模组名称、版本、作者、描述等信息。
* **一键启用/禁用**: 通过文件重命名（`.jar` ↔ `.jar.disabled`）切换模组状态，无需手动操作文件。
* **缺失依赖检测**: 自动扫描已启用模组的依赖声明，标记缺失依赖并告警。

### 🔗 Mod 依赖管理与可视化 (v1.0.5 新增)

提供四种视图切换，全方位展示模组之间的依赖关系：

* **列表视图**: 默认视图，以列表形式展示所有模组，展开可查看依赖详情。
* **依赖图谱 (文本列表)**: 以文本树形结构展示每个模组的依赖关系，标注缺失/已忽略状态。
* **力向图 (Force-Directed Graph)**: 基于 **d3.js v7** 的力导向物理仿真图：
    * 模组为圆角矩形节点，依赖为圆形节点，连线表示依赖方向。
    * 支持鼠标拖拽节点、滚轮缩放、平移画布。
    * 仿真稳定后自动缩放适配所有节点到视口中央。
    * 提供"重置"按钮恢复初始布局。
    * 颜色编码：绿色（正常）、红色（缺失）、蓝色（缺失但已忽略）。
* **网格图 (Grid Graph)**: CSS Grid 卡片布局，每个模组一张卡片，卡片内展示依赖标签药丸。
* **OS 级全屏**: 力向图和网格图提供全屏按钮，调用 pywebview 系统级全屏（非浏览器 requestFullscreen），适合大量模组时使用。
* **搜索过滤**: 四种视图均支持实时搜索过滤，搜索结果跨视图同步。
* **忽略缺失依赖**: 对已知可安全忽略的缺失依赖，可标记"确认可运行"（ignore），标记后该依赖从红色变为蓝色。可随时撤销。忽略数据保存在 `launcher_settings.json` 的 `mod_dep_ignores` 字段。

### ⚡ Mod 批量启用/禁用 (v1.0.5 新增)

* **批量模式**: 工具栏提供"批量操作"按钮，开启后进入多选模式。
* **多选操作**: 每个模组卡片出现选择复选框，支持逐个勾选、全选、反选。
* **一键批量操作**: 选定后点击"启用所选"或"禁用所选"，一次 API 调用完成所有操作。
* **结果汇总**: 返回操作结果摘要（总数 / 成功 / 失败），失败项附带具体原因。
* **磁盘状态回读**: 操作完成后自动刷新列表，确保界面状态与磁盘真实文件状态一致。

### ⚡ Mod 冲突规则提示引擎 (v1.0.5 新增)

* **本地规则文件**: 读取更新器同目录下的 `conflict_rules.json`，管理员/社区可维护已知冲突规则。
* **自动匹配**: 每次加载模组列表时，后端检查当前已启用模组是否命中规则。对于多模组规则（如 incompatible 类型），仅当规则中**所有**模组都已启用时才触发。
* **可视化提示**: 命中冲突的模组卡片显示 ⚡ 图标，展开详情可看到每条规则的 ID、说明和严重级别。
* **严重级别颜色**: `error`（红色）、`warning`（橙色）、`info`（蓝色）。
* **容错设计**: 规则文件不存在或格式错误时，模组列表正常显示，不会报错。

### 🗂️ 存档管理与 NBT 编辑器 (v1.0.7 新增)

在更新器中直接管理 Minecraft 存档，并通过独立窗口编辑任意 `.dat` / `.dat_old` NBT 文件：

* **存档列表**:
    * 自动扫描 `saves/` 目录，展示所有存档文件夹。
    * 解析每个存档的 `level.dat`，读取世界名称（LevelName）、游戏模式、种子（RandomSeed）、最后游玩时间（LastPlayed）等元数据。
    * 元数据解析基于自研 `NbtIO` 二进制解析器，纯 Python 标准库实现，不依赖任何第三方 NBT 库。
    * 支持搜索过滤和按版本分类筛选。
* **存档操作**:
    * 备份：将选中的存档复制到 `.save_backups/` 目录，备份以时间戳命名。
    * 删除：二次确认删除存档文件夹（移至回收站或直接删除）。
    * 导入：选择外部文件夹导入为新存档。
* **独立 NBT 编辑器窗口**:
    * 点击存档卡片的"编辑 NBT"按钮，打开独立 pywebview 窗口（1100×750，可调整大小）。
    * 窗口由独立模块 `TCYNBTeditor.py` + `TCYNBTeditor.html` 驱动，与主程序代码完全分离。
    * 采用 **VSCode 风格布局**：左侧文件树浏览器 + 右侧 NBT 编辑器。
* **文件树浏览器（左侧面板）**:
    * 递归扫描存档文件夹内所有子目录，自动过滤仅显示包含 `.dat` / `.dat_old` 文件的文件夹。
    * 文件夹可展开/折叠，文件夹优先排列，文件按字母排序。
    * 点击文件即在右侧编辑器中加载。当前打开的文件高亮显示。
* **NBT 树编辑器（中央面板）**:
    * **虚拟滚动架构**：仅渲染视口内可见行（ROW_HEIGHT × 行数），一个包含数千节点的 `level.dat` 也不会卡顿。
    * **搜索功能**：实时搜索 NBT 键名和标量值，匹配节点高亮显示，自动展开其所有父节点。支持上/下导航（F3 / Shift+F3 / Enter），显示 `n/m` 匹配计数。
    * **全部 13 种 NBT 标签类型**：Byte、Short、Int、Long、Float、Double、ByteArray、String、List、Compound、IntArray、LongArray，每种类型以独立彩色徽章区分。
    * **Long 值精度安全**：NBT Long（64 位整数）在 Python↔JS 传输时序列化为字符串，避免 JavaScript `Number.MAX_SAFE_INTEGER`（2⁵³-1）溢出导致种子等大数值失真。
    * **编辑模式**：点击"编辑模式"按钮开启，标量值变为可编辑输入框，标签名可直接重命名（Compound 子节点），容器节点可添加/删除子节点、修改类型。
    * **撤销/重做**：操作栈支持最多 200 步撤销，快捷键 Ctrl+Z / Ctrl+Y，覆盖编辑值、重命名、添加、删除、排序等所有操作。
    * **剪切/复制/粘贴**：支持跨节点复制粘贴（Ctrl+C/X/V），粘贴到 Compound 自动处理重名，粘贴到 List 自动匹配 list_type。
    * **右键上下文菜单**：右键任意节点弹出操作菜单（复制、剪切、粘贴、重命名、删除、按名称/类型排序、添加子节点），需要编辑模式的操作自动灰显。
    * **标签排序**：右键 Compound 节点可按名称或按类型排序子节点，支持撤销。
    * **保存 / 另存为**：编辑后点击"保存"写回原文件（gzip 压缩 NBT 二进制），或"另存为"（Ctrl+Shift+S）选择新路径保存。
    * **展开/折叠全部**：工具栏提供一键展开全部节点和折叠全部节点的快捷按钮。
    * **查找并替换**：Ctrl+H 展开替换栏，支持替换当前匹配/全部替换（仅标量值），每次替换记录 undo。
    * **SNBT 复制**：右键任意节点 → "复制为 SNBT"，将节点序列化为 Minecraft SNBT 文本格式复制到剪贴板。
    * **跨级拖拽移动**：编辑模式下可拖拽节点在树中任意移动——拖到行上方/下方插入同级，拖到容器中间 50% 区域移入内部；**拖向左侧**可按缩进级别逐级上移父容器（每 20px 一级），状态栏实时提示"上移 N 级"。
    * **编辑操作按钮**：工具栏提供复制/剪切/粘贴/删除/重命名/添加/排序实体按钮（根据选中节点类型自动启用/禁用），点击行即可选中节点。
    * **帮助弹窗**：顶栏"帮助"按钮弹出完整快捷键列表和功能说明，底部标注开发者 [GitHub @KanameMadoka520](https://github.com/KanameMadoka520)。
    * **未保存提示**：关闭有未保存修改的标签页或窗口时弹出确认对话框，防止误操作丢失编辑。
* **多标签页**:
    * 同时打开多个文件，每个标签独立保存完整编辑状态（tree、expanded、undo/redo、搜索、hex 视图等）。
    * 标签页可拖拽排序，有未保存修改的标签显示 ● 标记。
* **.mca 区域文件支持**:
    * 打开 `.mca` 文件显示 32×32 区块网格（Anvil 格式），颜色区分空区块/有数据/已选中。
    * 点击区块加载其 NBT 数据（支持 gzip/zlib/未压缩三种压缩类型），进入正常 NBT 编辑视图。
* **数组 hex 视图**:
    * ByteArray/IntArray/LongArray 展开后可点击 HEX 按钮切换十进制/十六进制显示（byte 2 位、int 8 位、long 直接转换）。
* **JSON 导出/导入**:
    * 导出：将 NBT 树转为可读 JSON（`{_type, _name, _value}` 格式）保存到文件。
    * 导入：从 JSON 文件加载并转换为 NBT 树，自动创建新标签页进入编辑模式。
* **拖放文件打开**:
    * 将 `.dat` / `.dat_old` / `.mca` 文件直接拖入编辑器窗口即可打开（覆盖层提示）。
* **操作日志与编辑会话追踪**:
    * 编辑器底部日志面板实时记录所有操作：修改值、重命名、添加、删除、排序、拖拽移动、文件打开/保存等。
    * 每条操作显示时间、类型（彩色标签）、文件名、可读路径描述（如 `root > Data > Player > Score`）。
    * **编辑会话系统**：每个文件的编辑生命周期为一个会话（session），由唯一 `sid` 标识：
      * 首次修改时自动创建 `▶ 开始编辑会话`
      * 保存 → `■ 结束会话: ✔ 已保存`
      * 关闭标签确认丢弃 → `■ 结束会话: ✖ 已取消`，该会话内所有操作条目变为半透明删除线（表示修改已作废）
      * 关闭窗口时自动结束所有未保存会话为"已取消"
      * 多文件并发编辑时各会话按文件路径独立追踪，互不干扰
    * 日志实时写入磁盘 `NBTEditor-log/nbt_session_YYYYMMDD.log`（按天分文件），取消时追加 `[CANCEL_BATCH]` 标记。
    * 面板支持清空和手动导出（完整日志含取消标记）。
* **大纲导航面板（右侧面板）**:
    * 展示文件中所有 Compound/List 结构节点的大纲（类似 VS Code Outline），显示类型徽章 + 名称 + 子节点数量。
    * 点击大纲项自动展开所有父节点并滚动定位到对应位置。
    * 工具栏"大纲"按钮切换面板显示/隐藏。
* **缩放**：编辑区和大纲面板支持 Ctrl+滚轮缩放（50%-200%），工具栏显示当前百分比按钮一键重置。
* **技术架构 — 代码分离**:
    * `TCYNBTeditor.py`（后端）：包含 `NbtIO` 类（二进制 NBT 解析/写入）、`NbtEditorApi` 类（pywebview JS API：打开文件、保存文件、扫描文件夹）、`open_nbt_editor()` 函数（创建独立窗口）。
    * `TCYNBTeditor.html`（前端）：包含 Catppuccin 风格深色主题 CSS + 完整的文件树渲染、NBT 树懒渲染、搜索、编辑功能的 JavaScript 代码。
    * 主程序通过 `from TCYNBTeditor import NbtIO, open_nbt_editor` 导入，存档列表解析 `level.dat` 时复用 `NbtIO.read()`。
    * `build.py` 将 `TCYNBTeditor.html` 打入 EXE（`ADDED_DATA`），并通过 `--hidden-import=TCYNBTeditor` 确保 PyInstaller 包含 Python 模块。

### 🌐 服务器状态 / 内置浏览器 (v1.0.7 新增)

无需打开外部浏览器，直接在更新器内查看服务器各线路状态，同时可作为轻量内置浏览器浏览任意网页：

* **本地反向代理**:
    * 原理：首次进入页面时，后端在 `127.0.0.1` 上随机端口启动一个本地 HTTP 反向代理服务，将请求转发至目标网站，并自动剥除 `Content-Security-Policy`、`X-Frame-Options` 等限制性响应头，使 iframe 可以正常嵌入原本禁止嵌入的页面。
    * 使用 `ThreadingTCPServer` 支持并发请求（CSS/JS/图片等并行加载不会串行阻塞）。非 HTML 资源透传 `Accept-Encoding` 实现 gzip 压缩传输；HTML 请求以未压缩方式获取以便注入导航拦截脚本。
    * 代理目标可动态切换：用户在地址栏输入新网址后，后端 `set_proxy_target()` 实时更新转发目标，iframe 重新加载即可访问新网站，无需重启代理。
    * 代理线程为 daemon 线程，主程序退出（含异常退出/崩溃）时自动终止，不会残留。
    * 界面显示当前代理占用的端口号、运行状态，以及数据流说明（实际数据来源为本地端口）。
* **内置浏览器功能**:
    * 地址栏：显示当前反向代理正在转发的**真实目标网址**（如 `https://server.tcymc.space/about`），而非本地代理地址。可手动输入任意网址并回车跳转。
    * 🏠 首页按钮：一键返回 `server.tcymc.space` 服务器状态页面。
    * 🔄 刷新按钮：刷新当前正在浏览的页面。
    * **链接拦截与自动跟随**：代理在 HTML 页面中自动注入导航拦截脚本，当用户点击指向外部域名的链接时，脚本通过 `postMessage` 通知父窗口，前端自动切换反代目标并重新加载，地址栏同步更新。站内相对链接跳转通过轮询 iframe 路径实时同步到地址栏。
* **多种访问方式**:
    * 内嵌 iframe（默认方式，基于反向代理）。
    * "打开独立窗口"：以 pywebview 独立窗口打开（不依赖反向代理，作为备用方案）。
    * "在浏览器中打开"：使用系统默认浏览器打开原始网址。
* **安全与提示**:
    * 毛玻璃卡片样式的安全提示：不建议在内置浏览器中登录账号，登录信息保存在 WebView2 缓存中，无法像正规浏览器一样管理 Cookie 和密码。
    * 毛玻璃卡片样式的数据流说明：标注地址栏显示的是反代目标网址，实际数据传输全部经由 `127.0.0.1:端口` 完成。
* **主动关闭**:
    * 提供"关闭内置网页服务"按钮，可随时手动停止反向代理。关闭后可一键重新启动。

## conflict_rules.json 冲突规则格式 (v1.0.5 新增)

该文件放在更新器同目录下（与 `launcher_settings.json` 同级），由管理员或社区维护：

```json
{
  "version": 1,
  "rules": [
    {
      "id": "RULE-001",
      "type": "incompatible",
      "mods": ["modA", "modB"],
      "description": "modA 和 modB 不兼容，同时启用会导致崩溃",
      "severity": "error"
    },
    {
      "id": "RULE-002",
      "type": "requires_version",
      "mods": ["modC"],
      "description": "modC 需要 1.20.1 版本的 Fabric API",
      "severity": "warning"
    },
    {
      "id": "RULE-003",
      "type": "note",
      "mods": ["modD"],
      "description": "modD 已知在低端显卡上可能导致帧率下降",
      "severity": "info"
    }
  ]
}
```

**字段说明**:

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | string | 规则唯一标识，如 `RULE-001` |
| `type` | string | 规则类型，如 `incompatible`、`requires_version`、`note`（仅用于分类展示） |
| `mods` | string[] | 涉及的 mod ID 列表（匹配 `fabric.mod.json` / `mods.toml` 中解析出的 `id`） |
| `description` | string | 冲突说明文本 |
| `severity` | string | 严重级别：`error`（红色）/ `warning`（橙色）/ `info`（蓝色） |

**匹配逻辑**:
- 多 mod 规则（如 `incompatible`）：**所有** `mods` 中的 mod 都已启用时才触发。
- 单 mod 规则：该 mod 已启用时即触发。
- 触发后，每个涉及的 mod 都会在列表中显示该规则的提示。

## 🛠️ 部署与适配指南——打造你自己的更新器

如果你想把这个更新器修改适配到你自己的 Minecraft 服务器（非 TCY 服务器），请严格按照以下步骤，从上到下逐一修改。

### 第一阶段：准备工作

1. **安装 Python 环境**

### 第二阶段：修改核心逻辑 (`TCYServer_MCUpdater.py`)

打开 `TCYServer_MCUpdater.py`，搜索并修改以下内容：

1. **修改目标版本文件夹名称 (必改!)**

* 找到代码开头部分的变量：`TARGET_VERSION_NAME = "异界战斗幻想"`
* **要做的事**: 把 `"异界战斗幻想"` 改成你整合包 `.minecraft/versions/` 下面那个文件夹的名字。
* *例子*: 如果你的整合包版本文件夹叫 `浩哥可爱捏`，就改成 `"浩哥可爱捏"`。

2. **修改初始版本号**

* 找到：`INITIAL_VERSION = "26.02.06.15.24"`
* **要做的事**: 改成你当前的客户端版本号。糖醋鱼个人建议你使用日期格式以免混淆。

3. **修改在线更新源 (最重要!)**

* 默认的更新源地址现在定义为文件顶部的常量，你需要修改它们（你需要一个可以托管 JSON 文件的网站，比如 GitHub Pages、Cloudflare Pages 或者你自己的服务器）。
* **搜索**: 文件顶部的 `DEFAULT_UPDATER_JSON_URL`

* **改为**: 你存放"更新器自身版本信息"的 JSON 链接。
* **搜索**: 文件顶部的 `DEFAULT_LATEST_JSON_URL`
* 原文：`DEFAULT_LATEST_JSON_URL = "https://tcymc.space/update/latest.json"`
* **改为**: 你存放"游戏内容更新信息"的 JSON 链接。
* **注意**: 用户也可以在"个性化设置 > 自定义更新源"中覆盖这些默认地址，无需修改代码。

4. **修改 User-Agent (可选)**

* 搜索 `User-Agent`，把里面的 `TCYClientUpdater/1.0` 改成你自己的名字，防止和原版混淆。

打开 `index.html`，这是用户看到的样子，需要把"糖醋鱼"的痕迹抹除。

1. **标题与品牌**

4. **默认镜像源**

* 搜索 `<input type="text" id="mirror-prefix-input"`：把 `placeholder` 和 `value` 里的 `https://gh-proxy.org/` 改成你推荐的镜像源。

5. **冲突规则文件 (v1.0.5 新增)**

* 如果你要使用冲突规则功能，在更新器同目录下创建 `conflict_rules.json`，按照上述格式填写你的服务器已知冲突规则。
* 如果不需要此功能，保持空规则即可（默认模板已提供）。

### 第四阶段：构建配置 (`build.py`)

打开 `build.py`，决定打包出来的 EXE 文件叫什么。

1. **修改程序名称**

* 找到：`EXE_NAME = "TCYClientUpdater-1.0.7"`
* **要做的事**: 改成你想要的程序名字，比如 `"终末地客户端更新器"`。

2. **额外数据文件 (v1.0.5 新增)**

* `ADDED_DATA` 列表定义了打包进 EXE 的额外文件：
  ```python
  ADDED_DATA = [("index.html", "."), ("lib/d3.min.js", "lib"), ("TCYNBTeditor.html", "."), ("jvm_advisor.js", "."), ("system_overview.js", ".")]
  ```
* `index.html` 是主程序前端页面，`lib/d3.min.js` 是力向图依赖，`TCYNBTeditor.html` 是 NBT 编辑器前端页面，`jvm_advisor.js` 和 `system_overview.js` 是外置前端功能模块。如果你添加了其他资源文件，也要在这里声明。
* PyInstaller 的 `--hidden-import=TCYNBTeditor` 确保 NBT 编辑器 Python 模块被打包进 EXE（因为是运行时 import，PyInstaller 静态分析无法自动发现）。

### 第五阶段：服务端搭建 (核心)

你的代码里填写的那些 JSON 链接（在第二阶段第3步），必须真实存在，并且内容格式要正确。你需要准备两个 JSON 文件上传到你的网站。

**1. 游戏内容更新 JSON (`latest.json`)**
格式必须如下：

```json
{
  "history": [
    {
      "version": "26.02.10.00.00",
      "date": "2026-02-10",
      "desc": "这里写更新日志：\n1. 修复了Bug\n2. 增加了Mod",
      "opt_update": false,
      "download_urls": {
        "cn": "https://你的网站/download/update_v2.zip",
        "global": "https://你的网站/download/update_v2.zip"
      }
    }
  ]
}
```

**2. 更新器自身更新 JSON (`Updater-latest.json`)**
格式必须如下：

```json
{
  "version": "1.0.7",
  "desc": "更新器自我升级：修复了闪退问题",
  "url": "https://你的网站/download/NewUpdater.exe"
}
```

### 第六阶段：打包发布

1. 确保 `icon.ico`, `background.png`, `index.html`, `lib/d3.min.js`, `TCYNBTeditor.html`, `TCYNBTeditor.py`, `jvm_advisor.js`, `system_overview.js` 都在文件夹里。
2. 在终端运行：

```bash

```

3. 等待运行完成，去新生成的 `dist` 文件夹里，把那个 `.exe` 文件发给你的玩家。
4. 如果你使用了冲突规则功能，记得把 `conflict_rules.json` 也一并分发。

### ⚠️ 适配避坑指南

1. **目录结构必须对**: 你的这个更新器有**强校验逻辑**。告诉你的玩家，必须把 `.exe` 放在和 `.minecraft` 文件夹**同级**的地方，否则不能正常完成客户端更新！
2. **HTTPS 问题**: 代码中已经添加了忽略 SSL 证书验证 (`ssl._create_unverified_context`)，所以即使用自签名的证书一般也没问题，但最好还是用正规的 HTTPS 链接。
3. **不要删 `index.html`**: 虽然打包脚本把它打进去了，但在打包前，它必须存在。
4. **不要删 `lib/d3.min.js`**: 力向图渲染依赖此文件，打包时会自动打入 EXE。
5. **不要删 `TCYNBTeditor.html` 和 `TCYNBTeditor.py`**: NBT 编辑器依赖这两个文件。`TCYNBTeditor.html` 通过 `ADDED_DATA` 打入 EXE，`TCYNBTeditor.py` 通过 `--hidden-import` 打入 EXE。
6. **不要删 `jvm_advisor.js` 和 `system_overview.js`**: JVM 调优与系统概览页面依赖这两个外置前端模块，打包时会通过 `ADDED_DATA` 自动带入 EXE。

## 故障排查

* **启动时弹出"目录结构校验失败"**:
* 原因：更新器未检测到 `.minecraft/versions/异界战斗幻想` 路径。
* 解决：请确保将更新器 EXE 放在游戏整合包的根目录下（和 `.minecraft` 文件夹在同一层）。
* **更新器改名后无法自动更新**:
* 现已修复。本版本支持任意改名（包括含 Emoji 的名字），更新后会自动保持您修改后的名字。
* **为什么有旧版本提示?**:
* 如果你之前跳过了某个可选更新，系统会一直将其保留在更新队列中，属于"补票机制"的正常现象。
* **CN线路下载失败**:
* 请在"个性化设置"中更改"GitHub 镜像加速前缀"（推荐 `https://gh-proxy.org/`）。v1.0.3 已支持自动测速选择最快镜像。
* **无法连接主站获取版本信息**:
* 程序会自动依次尝试官网、GitHub 备用地址、GitHub 加速前缀地址，通常无需手动干预。若所有地址均失败，弹窗会列出已尝试的全部地址。此时可在"个性化设置 > 自定义更新源"中填写管理员提供的备用 JSON 地址，或检查"GitHub 镜像加速前缀"是否有效。留空则使用默认地址，点击"恢复默认更新源"可一键恢复。
* **程序未响应**:
* 查看同级目录下的 `launcher_debug.log` 或 `CRASH_IMPORT.txt` 获取堆栈信息。
* **更新失败后如何回滚** (v1.0.3 新增):
* 更新失败会自动回滚。如需手动回滚，在更新页面的"备份管理"区域选择备份版本点击"回滚"。
* **日志导出** (v1.0.3 新增):
* 更新页面点击"导出日志"按钮，选择保存位置即可导出完整运行日志。
* **更新摘要弹出后超时** (v1.0.5 新增):
* 更新预览 token 有效期为 600 秒（10 分钟）。如果超时，请重新点击更新按钮获取新的摘要。
* **Mod 依赖图谱显示异常** (v1.0.5 新增):
* 力向图依赖本地 `lib/d3.min.js`。如果打包后力向图无法渲染，请检查 `build.py` 的 `ADDED_DATA` 是否包含 `("lib/d3.min.js", "lib")`。
* **冲突规则不生效** (v1.0.5 新增):
* 确认 `conflict_rules.json` 放在更新器 EXE 同目录下（与 `launcher_settings.json` 同级）。
* 确认规则中的 `mods` 数组使用的是 mod 的 `id`（来自 `fabric.mod.json` 或 `mods.toml`），而非文件名。
* **NBT 编辑器窗口打不开** (v1.0.7 新增):
* 检查 `TCYNBTeditor.html` 是否在打包目录中。如果使用源码运行（`python TCYServer_MCUpdater.py`），确保 `TCYNBTeditor.html` 和 `TCYNBTeditor.py` 在同级目录。
* 如果使用打包后的 EXE，确保 `build.py` 的 `ADDED_DATA` 包含 `("TCYNBTeditor.html", ".")` 且 `--hidden-import=TCYNBTeditor` 已添加。
* **NBT 文件解析失败** (v1.0.7 新增):
* `NbtIO` 仅支持 Java 版 NBT 格式（big-endian），不支持基岩版 NBT（little-endian）。
* 仅支持 `.dat` 和 `.dat_old` 后缀的文件。
* 支持 gzip 压缩和未压缩两种格式（自动检测）。
* **服务器状态页面加载失败** (v1.0.7 新增):
* 本功能依赖本地反向代理。如果显示"反向代理启动失败"，检查是否有防火墙/安全软件阻止了本地端口监听。
* 反向代理仅监听 `127.0.0.1`（本机回环地址），不对外网开放，不存在安全风险。
* 如果加载缓慢，这与 `server.tcymc.space` 的网络延迟有关，可尝试刷新或使用"打开独立窗口"/"在浏览器中打开"作为替代方案。
* 如需手动关闭代理服务，点击"关闭内置网页服务"按钮即可。

* **启动自动更新体验改造（已完成）**：启动时自动更新检测已改为静默模式，并在程序全局页面顶端显示“灵动岛风格”顶部悬浮提示；支持加载中、检测到更新（点我查看详情）、检测失败（点我重试）、无更新自动消失等状态。当前悬浮提示已接入主题色与个性化风格，并增加了内容宽度平滑伸缩、点击压缩反馈与更顺滑的过渡动画；旧的连接结果弹窗分支已移除，启动与手动检查更新统一走顶部提示 + 版本详情链路。

### 远期规划

* **多语言支持 (i18n)**：将 UI 文案抽离为语言包，方便其他服务器适配不同语言。
* **系统托盘最小化**：关闭窗口时缩到系统托盘而非直接退出，方便后台等待下载完成。
* **下载限速**：提供可配置的带宽限制，避免更新时占满用户网络。
* **SSL 证书验证优化**：仅对特定内网地址跳过 SSL 验证，其他地址走正常验证流程。
* **更新包签名验证**：对下载的 ZIP 包进行数字签名校验，防止中间人攻击。
* **自动错误上报**：崩溃或更新失败时可选地将日志摘要发送到服务端，方便排查问题。
